#### 3a. `analyzeApplianceLifecycle(householdId: string): Promise<InsightResult[]>`

For each active inventory item in the household:
1. Calculate current age from `purchaseDate`
2. Look up the item's category (HVAC, APPLIANCE, etc.) in the network aggregate data using `getApplianceLifespanAnalytics(item.category, { brand: item.brand, region })` from `aggregate-analytics.ts`
3. If the aggregate returns data (meets k-anonymity), compare the item's age to the network median lifespan
4. Generate insights at these thresholds:
   - **Item age > 80% of median lifespan** → category `REPLACEMENT_FORECAST`, confidence 70-85. Title: `"Your [brand] [name] is approaching end of life"`. Summary explains age vs. network average, estimated remaining years, and suggests budgeting for replacement. Include `estimatedReplacementYear`, `networkMedianLifespan`, `currentAgeYears`, `estimatedReplacementCostCents` in data.
   - **Item age > 100% of median lifespan** → category `REPLACEMENT_FORECAST`, confidence 85-95. Title: `"[name] is past typical lifespan — plan replacement"`. Higher urgency messaging. Include marketplace search link in data: `actionUrl: "/marketplace?category=HVAC&search=water+heater"`.
   - **Item age within 10-30% of median AND has failure patterns from network for its age bucket** → category `APPLIANCE_LIFESPAN`, confidence 60-75. Title: `"Heads up: [category] units like yours often need [common failure] around this age"`. Proactive prevention suggestion.
5. If aggregate data is insufficient (doesn't meet k-anonymity), fall back to industry-standard lifespan estimates:

#### 3b. `analyzeConsumableNeeds(householdId: string): Promise<InsightResult[]>`

1. Query `householdConsumableTracking` joined with `applianceConsumables` and `inventoryItems` for this household
2. For each tracked consumable, calculate days until `nextDueDate`
3. Generate insights:
   - **Due within 7 days** → category `CONSUMABLE_REMINDER`, confidence 90, priority HIGH equivalent. Title: `"Time to replace [consumableName] for your [itemName]"`. Summary: `"Your [brand] [model] [consumableSize] filter is due [in X days / this week / today]. [If not stocked, order one through hndld →]"`. Include `actionUrl` linking to marketplace search or affiliate URL. Include `consumableSize`, `partNumber`, `estimatedCost`, `daysDue`, `inventoryItemId` in data.
   - **Due within 14-30 days** → confidence 80, MEDIUM priority. Gentler reminder: `"Your [consumableName] is due in [X] days — check if you have a replacement ready"`.
   - **Overdue (past nextDueDate)** → confidence 95, HIGH priority. `"[consumableName] replacement is [X days] overdue"`. Stronger action language.
4. Also: For inventory items that have `serviceIntervalDays` set but are NOT yet tracked in `householdConsumableTracking`, auto-match against `applianceConsumables` by category+brand. Generate a one-time SUGGESTION insight: `"We can track [consumable] replacements for your [item] — want to set up reminders?"` with `actionUrl: "/inventory/[itemId]"`.

#### 3c. `analyzeWarrantyActions(householdId: string): Promise<InsightResult[]>`

1. Query inventory items where `warrantyExpires` is within 90 days from now
2. For items expiring within 30 days → category `WARRANTY_ACTION`, confidence 90. Title: `"Warranty on your [name] expires [date]"`. Summary suggests: check for any issues to claim before expiry, consider extended warranty, document current condition with photos.
3. For items where warranty just expired (within last 30 days) AND the item's age is approaching the network failure zone → category `WARRANTY_ACTION`, confidence 75. Summary: `"Your warranty just ended and [item] is entering its most failure-prone years. Consider a home warranty or setting aside a repair fund."`. Include estimated replacement cost from network data if available.
4. For items with active warranties that have had recent service events with `wasWarrantyService = false` that could have been warranty-covered → category `WARRANTY_ACTION`, confidence 65. `"Your recent [service type] on [item] might have been covered by your active warranty. Check with [warrantyProvider]."`.

#### 3e. `generateAllPredictiveInsights(householdId: string): Promise<InsightResult[]>`

Orchestrate all the above with `Promise.all` and safe error handling (same pattern as `generateAllInsights` in `home-intelligence.ts`):

```typescript
export async function generateAllPredictiveInsights(householdId: string): Promise<InsightResult[]> {
  const safeRun = (name: string, fn: () => Promise<InsightResult[]>) =>
    fn().catch(e => {
      logger.error(`[PredictiveMaintenance] ${name} failed`, {
        error: e instanceof Error ? e.message : String(e), householdId,
      });
      return [] as InsightResult[];
    });

  const [lifecycle, consumables, warranty] = await Promise.all([
    safeRun("Lifecycle", () => analyzeApplianceLifecycle(householdId)),
    safeRun("Consumables", () => analyzeConsumableNeeds(householdId)),
    safeRun("Warranty", () => analyzeWarrantyActions(householdId)),
  ]);

  const combined = [...lifecycle, ...consumables, ...warranty];

  // AI insight only if under 8 insights and AI is available
  if (combined.length < 8 && getActiveProvider() !== "NONE") {
    try {
      const items = await db.select().from(inventoryItems)
        .where(and(eq(inventoryItems.householdId, householdId), eq(inventoryItems.isActive, true)));
      const [detail] = await db.select().from(householdDetails)
        .where(eq(householdDetails.householdId, householdId)).limit(1);
      const aiInsight = await generateAIPredictiveInsight(householdId, items, detail || null);
      if (aiInsight) combined.push(aiInsight);
    } catch (e) {
      logger.error("[PredictiveMaintenance] AI insight failed", {
        error: e instanceof Error ? e.message : String(e), householdId,
      });
    }
  }

  return combined.sort((a, b) => b.confidence - a.confidence);
}
```
