Replit Agent Prompt: Implement Custom Icon System for hndld
You are a senior frontend engineer implementing a custom branded icon system for hndld, a luxury household concierge platform. The app uses React + Vite + TypeScript + TailwindCSS + shadcn/ui. Replace all generic Lucide icons with a custom dual-tone icon system that matches the brand.

CONTEXT
hndld currently uses Lucide React icons throughout the app (bottom nav, page headers, cards, buttons, empty states). We're replacing them with a custom SVG icon set that uses two-tone navy + champagne gold coloring. Lucide remains as a fallback for minor UI elements (chevrons, close buttons, etc.), but all feature-level icons get the custom treatment.
Brand palette:

--hndld-charcoal: #2A3444   (navy charcoal — secondary strokes)

STEP 1: Create the Icon Component System
Create: client/src/components/icons/hndld-icons.tsx
Build a typed React icon component that renders inline SVGs. Each icon should:

Accept size prop (default: 24), className prop, and all standard SVG props
Use currentColor for the navy strokes so they inherit from parent text color
Use a dedicated accentColor prop (default: #C9A96E) for the gold accent elements
Use 1.75px stroke width, round linecap, round linejoin consistently
Have a viewBox of 0 0 24 24 with no fill on stroke elements

tsximport React from "react";

interface HndldIconProps extends React.SVGProps<SVGSVGElement> {
  size?: number;
  accentColor?: string;
}

const defaultProps: HndldIconProps = {
  size: 24,
  accentColor: "#C9A96E",
};

// Wrapper component for consistency
const IconWrapper: React.FC<HndldIconProps & { children: React.ReactNode }> = ({
  size = 24,
  accentColor = "#C9A96E",
  className,
  children,
  ...props
}) => (
  <svg
    xmlns="http://www.w3.org/2000/svg"
    width={size}
    height={size}
    viewBox="0 0 24 24"
    fill="none"
    strokeWidth="1.75"
    strokeLinecap="round"
    strokeLinejoin="round"
    className={className}
    {...props}
  >
    {children}
  </svg>
);

interface IconProps extends HndldIconProps {
  name: HndldIconName;

export function Icon({ name, ...props }: IconProps) {
  const IconComponent = hndldIcons[name];
  return <IconComponent {...props} />;
}

VISUAL REFERENCE
The icons follow this exact style:

Geometric precision — clean bezier curves, no hand-drawn quality
1.75px stroke weight — consistent across all icons, softer than standard 2px
Rounded caps and joins — every line ending and corner is rounded
Two-tone execution — navy (#1A2332) for structure, champagne gold (#C9A96E) for one accent element per icon
24x24 grid — designed to be pixel-clear at this base size
Style benchmarks — Phosphor Icons precision, Linear app aesthetic, Headspace warmth


WHAT NOT TO DO

Do NOT use filled/solid icon styles — these are outline icons with selective accent fills only
Do NOT add drop shadows, glows, or gradients to the icons
Do NOT make the gold accent elements too large — they should be subtle (a checkmark, a dot, a small heart)
Do NOT change the stroke width per icon — 1.75px everywhere
Do NOT use Lucide as a "close enough" substitute for the feature icons — the whole point is brand differentiation
Do NOT forget the active/inactive state distinction in navigation — inactive icons should feel muted
